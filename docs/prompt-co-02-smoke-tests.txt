# 4. 現状理解（コードから読み取った事実）

## 既存テストの現状
- `TestHelpers.swift` — `TestModelContainer` と `STTCoreTests` を含む
  - **重要**: iOS 26.2 Simulator で SwiftData ModelContainer を複数作成すると EXC_BREAKPOINT が発生するバグあり
  - そのため ModelContext を使ったテストは不可。@Model オブジェクトのプロパティテストのみ実行
  - `STTCoreTests`: OnDeviceTranscriptionTimeoutError のエラーメッセージテスト2件
- `FileDetailViewModelTests.swift` — formatDate/formatDuration/formatTime + 初期状態 + audioFile 保持テスト
- `AudioFileTests.swift` / `TodoItemTests.swift` / `TranscriptTests.swift` — モデルのプロパティテスト
- `KnowledgePipelineTests.swift` — KnowledgeIndexingService のテスト

## テストの制約
- **SwiftData ModelContext が使えない** — @Model オブジェクトの直接テストのみ可能
- テストは `@Test` マクロ（Swift Testing）を使用
- `@MainActor` が必要なテストには `@MainActor` を付与

## テストすべき内容
1. **STT preflight 関連**（CL-01）:
   - `STTErrorMapper` のマッピングロジック
   - `OnDeviceTranscriptionTimeoutError` のプロパティ（既存テストあり、拡張）
   - `SpeechAnalyzerFeatureFlag` のデフォルト値
   - `LocalTranscriptionError` のエラーメッセージ

2. **KnowledgeIndexingService 静的メソッド**（CL-04 関連、既存インデックス）:
   - `chunkText(_:targetLength:hardLimit:)` の分割ロジック
   - `extractKeywords(from:limit:)` のキーワード抽出
   - `normalizedIndexableText(_:)` の正規化
   - ※ これらは `nonisolated static` メソッドなので ModelContext 不要

3. **ProcessingJob モデル**（CL-06）:
   - プロパティのデフォルト値（ModelContext 不要なら）
   - `canRetry` ロジック（retryCount < maxRetries）
   - ヘルパーメソッドの動作（markStarted, updateProgress, markCompleted, markFailed）

4. **FileDetailViewModel**（CL-03）:
   - 既存テストにタブ関連のテストを追加（selectedTab 等）

# 5. 実装要件

### STTPreflightTests（新規）
1. `STTErrorMapper` の全ケースのマッピングテスト
2. `LocalTranscriptionError` の各ケースの `errorDescription` テスト
3. `SpeechAnalyzerFeatureFlag.isEnabled` のデフォルト値確認（テスト実行時に他テストと独立させる）
4. `AIError` の各ケースのエラーメッセージテスト

### KnowledgeIndexingTests（既存拡張）
1. `KnowledgeIndexingService.chunkText` のテスト:
   - 空文字 → 空配列
   - 短い文字（targetLength 未満）→ 1要素
   - 長い文字 → 複数要素に分割、各要素が hardLimit 以下
   - 改行区切りのテキストの分割
2. `KnowledgeIndexingService.extractKeywords` のテスト:
   - 空文字 → 空配列
   - 英語のキーワード抽出
   - 日本語のキーワード抽出
   - limit パラメータの動作確認
3. `KnowledgeIndexingService.normalizedIndexableText` のテスト:
   - nil → nil
   - 空文字 → nil
   - 正常テキスト → トリム済みテキスト

### ProcessingJobTests（新規）
※ SwiftData ModelContext が使えないため、@Model クラスの直接インスタンス化も制限がある。
可能な範囲でテスト:
1. `canRetry` のロジックテスト:
   - retryCount=0, maxRetries=1 → true
   - retryCount=1, maxRetries=1 → false
   - retryCount=0, maxRetries=3 → true
2. もし @Model の直接インスタンス化が不可の場合は、ロジックを static メソッドに切り出してテストする（ただし「ついでリファクタ」にあたるため、最小限に留める）

### FileDetailViewModelTests（既存拡張）
1. `FileDetailTab` enum のケース確認テスト:
   - `.summary / .transcript / .memo` の3ケースが存在
2. タブ表示に関連する ViewModel プロパティの確認テスト
