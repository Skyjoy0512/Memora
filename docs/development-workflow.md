# Memora 並行開発ワークフロー

## 概要

Memora プロジェクトを Windows と macOS の両方で並行開発するためのワークフローを定義します。

## 基本的な役割分担

### Windows 側（Claude Code）
- **メインの役割**: 設計・設計・ドキュメント管理
- **主な作業**:
  - 実装計画の立案・調整
  - 仕様書の確認・補足
  - ドキュメントの作成・更新
  - GitHub 管理（ブランチ、プル、プッシュ）
  - コードレビュー（macOS 側で実装されたコード）
  - 問題の調査・解決
  - テスト計画の立案

### macOS 側（Xcode）
- **メインの役割**: 実装・テスト・デバッグ
- **主な作業**:
  - Xcode プロジェクト作成
  - Swift コードの実装
  - テストの実行
  - デバッグ
  - GitHub への push

## 開発フロー

### フェーズ 1: 基本構築（現在進行中）

#### Windows 側での作業
1. **仕様書の確認**
   - 詳細な仕様書の内容を理解
   - 実装範囲の明確化
   - 技術的な課題の事前確認

2. **計画の立案**
   - 実装計画の作成
   - フェーズごとのタスク分解
   - 依存関係の確認
   - テスト計画の立案

3. **GitHub 管理**
   - ブランチ戦略の検討
   - 必要に応じてブランチ作成
   - コミットメッセージの管理

4. **ドキュメント作成・更新**
   - 開発ステータスの記録
   - 問題・解決のドキュメント化
   - ビルド・テスト環境の説明

5. **コードレビュー支援**
   - macOS 側で実装されたコードのレビュー
   - ベストプラクティスへの従ってチェック
   - リファクタリングの提案
   - 問題点の指摘

#### macOS 側での作業
1. **Xcode プロジェクト作成**
   - 新規プロジェクトの作成
   - iOS 18+ のターゲット設定
   - Swift 6 の設定
   - Strict Concurrency の有効化

2. **TCA 依存の追加**
   - swift-composable-architecture のパッケージ追加
   - SPM での解決

3. **基本ディレクトリ構成の作成**
   - App/, Core/, Features/, DesignSystem/ ディレクトリ作成
   - 各ディレクトリのグループ作成

4. **SwiftData セットアップ**
   - モデル定義
   - SwiftDataStack の実装
   - Repository の実装

5. **デザインシステムの実装**
   - カラートークン
   - タイポグラフィ
   - コンポーネント

6. **基本的な UI 実装**
   - FilesListReducer
   - FilesListView
   - FilesRowView
   - RecordingReducer
   - RecordingView
   - AppReducer
   - TabView

7. **GitHub への push**
   - 実装完了後のコードをプッシュ
   - 定期的な同期

## コミュニケーション方法

### 定期的な同期

1. **Pull Request の作成**
   - 大きな変更の際は PR を作成
   - コードレビューを Windows 側で実施
   - macOS 側で確認後にマージ

2. **直接コミットの推奨**
   - 小さな修正は直接コミットでも可
   - ブランチ戦略に従う

3. **Issue による連携**
   - 課題や課題を Issue で管理
   - 状態の共有

## ドキュメント管理

### 開発ステータスの記録

#### docs/development-workflow.md
- ワークフローの定義
- 役割分担の明確化
- 連携方法の記述

### 進捗管理

#### タスク管理
- docs/todo.md を使用
- フェーズごとの進捗管理
- 完了したタスクのマーク

### 技術的な決定事項

#### アーキテクチャ
- TCA（The Composable Architecture）を採用
- iOS 18+, Swift 6, Strict Concurrency
- SwiftUI を主体とした UI 開発

#### 依存管理
- Swift Package Manager で統一管理
- swift-composable-architecture 1.17+
- AuthenticationServices（Apple 認証）

## 問題解決プロセス

### 報告事項

1. **問題の報告**
   - macOS 側で発生した問題を報告
   - 可能な解決案の提供

2. **協議**
   - 技術的な課題について話し合う
   - 設計の調整

3. **解決**
   - 合意形成後の実装
   - 解決方法の文書化

## テスト戦略

### 単体テスト
- Repository レベルのテスト
- Service レベルのテスト
- UI コンポーネントのテスト

### 統合テスト
- ユーザーストーリーに基づいたテスト
- 画面間のフロー確認

## 進捗管理の更新

### フェーズ 1 の状況
- **完了**:
  - ✅ 仕様書の分析
  - ✅ 基本構築の計画立案
  - ✅ Windows 側準備作業
  - ✅ GitHub Actions の追加

- **進行中**:
  - ⏳ macOS 側での Xcode プロジェクト作成
  - ⏳ 基本ディレクトリ構成の作成

## 注意点

- **並行開発の同期**: GitHub を通じて定期的な同期を行う
- **コード品質の維持**: ベストプラクティスに従う
- **ドキュメントの更新**: 進捗状況をリアルタイムに反映
- **コミュニケーション**: 問題が発生したら迅速に連携

## 次のステップ

1. **macOS 側**: Xcode プロジェクト作成から開始
2. **Windows 側**: ワークフロードキュメントの補足・更新
3. **GitHub**: CI/CD 環境の確認・調整
