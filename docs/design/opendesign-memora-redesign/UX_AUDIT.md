## 1. 監査サマリー

7つの観点で既存UIを評価した。各項目5点満点、合計35点満点。

| 観点 | スコア | 主な課題 |
|------|--------|----------|
| 情報設計 | 2/5 | ホームに詰め込みすぎ、情報の優先順位が不明瞭 |
| 操作導線 | 2/5 | キャプチャフローが多段モーダルで離脱リスク大 |
| 状態設計 | 1/5 | ローディング・空・エラーが粗く、ユーザーを迷わせる |
| 一貫性 | 3/5 | トークンは整備されているが画面間で適用にばらつき |
| 可読性 | 3/5 | タイプスケールは適切だが長文化で行間が不足 |
| アクセシビリティ | 3/5 | タップ領域・ラベルは良好だが状態通知が弱い |
| 日本語最適化 | 2/5 | 文言が説明的で長く、一部に英語が混在 |
| **合計** | **16/35** | — |

## 2. 画面別課題と改善方針

### 2.1 ホーム画面

#### 現状の課題

1. **3つのビューモードが複雑すぎる**
   - 「全ファイル」「プロジェクト」「ライフログ」の3モードが、画面タイトル部のドロップダウンでしか切り替えられない
   - ライフログの水平カレンダーは操作性が悪く、発見もしにくい
   - プロジェクトビューとファイルビューの関係が不明瞭

2. **検索の発見性が低い**
   - ヘッダー右上の虫眼鏡アイコンのみで、使われにくい
   - 検索中はタイトルが検索バーに置き換わり、文脈を見失う

3. **「未接続」表示が常に目立つ**
   - デバイス未接続状態が目立ちすぎ、日常的な使用の妨げになる

4. **グループ化が時間ベースのみ**
   - 「今日」「今週」「以前」だけでは、蓄積が増えたときに破綻する

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | ホームを単一の「最近の記録」リストに統一。セグメントコントロールで「すべて」「お気に入り」「プロジェクト」を切り替え | 高 | 認知負荷 50%減 |
| 2 | 検索バーを常時表示（ヘッダー直下）に変更。スクロールで隠れるが、引き下げで表示 | 高 | 検索利用率 向上 |
| 3 | デバイス状態は設定画面のみに表示。ホームでは「録音可能」かどうかのみをアイコンで示す | 中 | 視覚ノイズ減 |
| 4 | 日付セパレーターを維持しつつ、スクロール位置に応じたフローティング日付表示を追加 | 中 | 長いリストでの位置把握が容易に |
| 5 | クイックアクション（最近のプロジェクト・よく使うテンプレート）をリスト上部に小さく表示 | 低 | 再利用の促進 |

### 2.2 キャプチャ導線（録音・取り込み）

#### 現状の課題

1. **3段階のモーダル画面が長すぎる**
   - 録音モーダル → 生成設定モーダル → 生成進捗モーダルの3段階は、ユーザーの目的「記録して保存する」に対して遠回り
   - 「スキップ」ボタンの存在が、生成が必須でないことを示唆しているのに、なぜ最初に選択させるのか

2. **録音中の操作が限定的**
   - ハイライト（ブックマーク）機能があるが、録音後にどう使われるのか不明瞭
   - 一時停止・停止・破棄の3操作のみで、メモを取るなどの併用操作がない

3. **FABメニューの情報設計が弱い**
   - 「録音開始」「インポート」「会議キャプチャー」の3項目だが、優先順位や説明がない
   - 「会議キャプチャー」は「準備中」で、なぜ表示されているのか

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | 録音フローを1画面に統合。録音中にタイトル入力・テンプレート選択を済ませられる | 高 | 操作ステップ 60%削減 |
| 2 | FABメニューを「録音」（最優先・大）と「読み込み」（小）の2項目に整理 | 高 | 選択の迷いを排除 |
| 3 | 録音停止後は即座に保存し、文字起こしはバックグラウンドで自動開始 | 高 | ユーザーの待ち時間ゼロ |
| 4 | 録音画面にクイックメモ欄を追加（後で詳細を書くための手がかり） | 中 | 録音後の編集負担減 |
| 5 | 録音停止後の確認を「保存しました」の軽いトースト通知に変更（画面遷移しない） | 中 | フロー完了の明確化 |

### 2.3 ファイル詳細画面

#### 現状の課題

1. **サマリータブに情報を詰め込みすぎ**
   - メタ情報・チャプター・決定事項・次のアクション・添付・要約が1タブに
   - スクロール量が多く、目的の情報にたどり着くまでに時間がかかる

2. **文字起こしの視認性が低い**
   - 話者と発言の区別が弱い（話者名が小さく、タイムスタンプも小さい）
   - 長い文字起こしで現在位置を見失う
   - 再生中のセグメントハイライトが薄すぎる

3. **メモタブと他のタブの関係が不明瞭**
   - メモは「ファイルに対する自分のメモ」なのに、サマリー内の「決定事項」や「次のアクション」とどう違うのか
   - メモから写真添付への導線はあるが、使い道が想像しにくい

4. **要約生成が手動トリガー依存**
   - 「要約を再生成」ボタンを押さないと要約が作られない
   - AIモデルの選択が設定画面にあり、ここでは変えられない

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | サマリータブを「概要」に改名し、内容を3セクション（要約・要点・アクション）に整理 | 高 | 情報発見時間 50%短縮 |
| 2 | 文字起こしタブ：話者ごとの色分け、再生位置の明示的ハイライト、セグメント間の余白拡大 | 高 | 長文可読性 大幅向上 |
| 3 | メモタブを「自分のメモ」として独立させ、サマリーからの差分を明確に | 中 | 用途の明確化 |
| 4 | 文字起こし・要約は保存後自動実行。進捗はタブ内に小さく表示 | 中 | 手動操作の削減 |
| 5 | アクション項目をタスク画面と連携させ、チェックで完了にできる | 中 | タスク管理との統合 |

### 2.4 Ask AI 画面

#### 現状の課題

1. **会話の永続性がない**
   - スコープ（全体/プロジェクト/ファイル）を切り替えると会話がリセットされる（実装上）
   - 過去の会話履歴を振り返る手段がない

2. **ソース表示が不完全**
   - 回答の参照元が小さなテキストで表示されるが、タップしても詳細に飛べない
   - どの記録のどの部分を参照したかがわからない

3. **入力エリアの設計**
   - ファイル添付ボタンがあるが「準備中」
   - プレースホルダがグローバルスコープ時に「すべての記録から探す」で抽象的すぎる

4. **空状態の質問例が固定**
   - 3つの質問例が文脈に関係なく固定表示
   - ユーザーの実際の記録内容に基づいた提案がない

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | スコープ切替を会話上部のチップに。切替時に確認ダイアログを表示（会話がリセットされることを伝える） | 高 | データ消失の防止 |
| 2 | 参照元をタップ可能にし、該当ファイルの詳細画面に遷移 | 高 | 回答の信頼性向上 |
| 3 | 質問例を「最近の記録」に基づいて動的に生成（例：「先週の定例の決定事項は？」） | 中 | 発見と利用の促進 |
| 4 | 会話履歴一覧を追加（設定またはAsk AI内の履歴ボタンから） | 低 | 継続利用の促進 |
| 5 | 回答下部の「コピー」「タスク化」アクションを実装 | 中 | 実用性の向上 |

### 2.5 タスク画面

#### 現状の課題

1. **完全なモックデータで動作していない**
   - ハードコードされたタスクのみ。実際のファイルのアクション項目と連携していない
   - タスク追加はできるがファイルと結びつかない

2. **期限の概念が粗い**
   - 「期限切れ」「今日」「今後」の3区分のみ
   - 期日の設定・変更ができない

3. **完了済みタスクの扱い**
   - デフォルトで折りたたまれているが、完了した達成感が弱い

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | ファイル詳細の「アクション項目」から自動的にタスクを生成する導線を確立 | 高 | タスク利用率向上 |
| 2 | 期日設定UIを追加（日付ピッカー） | 中 | タスク管理の実用化 |
| 3 | 完了アニメーション（チェック時に一瞬だけ緑背景）を追加 | 低 | 達成感の演出 |

### 2.6 設定画面

#### 現状の課題

1. **ユーザー向け設定と開発者向け診断が混在**
   - 「開発者向け」セクション以下に Bridge 診断やフォントプレビューが露出している
   - 一般ユーザーには不要・混乱する情報

2. **多くの項目が「準備中」**
   - アカウント・デバイス・ストレージ・連携・言語の各項目が「未接続」「未設定」
   - 「準備中」のアラートが多用され、操作のたびに中断される

3. **設定の構造がフラットすぎる**
   - 10以上のセクションが同じ階層で並び、目的の設定を見つけにくい

#### 改善方針

| # | 改善 | 優先度 | 期待効果 |
|---|------|--------|----------|
| 1 | 開発者向け情報を Settings から分離し、専用のデバッグ画面に移動（一般ユーザーには非表示） | 高 | 設定画面の整理 |
| 2 | 「準備中」項目を減らし、利用可能な設定のみを表示 | 中 | 離脱防止 |
| 3 | 設定を「アカウント」「通知」「データ」「情報」の4グループに再編 | 中 | スキャンしやすく |

## 3. 横断的課題

### 3.1 状態表示

| 状態 | 現状 | 改善後 |
|------|------|--------|
| ローディング | 「ファイルを読み込み中」の汎用表示のみ | スケルトン画面（情報の形を予告） |
| 空（初回） | ファイルがありません + 説明文 | 最初の録音を促すイラスト + 大きな開始ボタン |
| 空（検索） | 一致するファイルがありません | 検索条件の緩和提案（「プロジェクト名でも検索できます」） |
| エラー | 技術的なエラーメッセージ | ユーザーに伝わる日本語 + 再試行ボタン |
| 保存中 | 表示なし（不安） | ヘッダーまたはタブバー付近の小さなインジケータ |
| 保存完了 | 画面遷移して暗黙的に通知 | 「保存しました」トースト + 次のアクション提案 |
| オフライン | 未対応 | 「オフライン」バナー + ローカル利用可能な機能の明示 |

### 3.2 ナビゲーション

| 課題 | 改善 |
|------|------|
| タブバーが4項目 + FABで混雑 | タブを「ホーム」「検索」「タスク」「設定」に再編。Ask AI はホームからアクセスできるように |
| ファイル詳細からの「戻る」がヘッダー左上のみ（片手操作で遠い） | スワイプバック対応（既存だが明示） + 「この記録について聞く」ボタンの位置調整 |
| 深い画面からのホーム復帰が複数タップ必要 | タブバータップでスタックのルートに戻る（標準挙動の明示的活用） |

### 3.3 空状態の設計

現状、空状態はすべてグレーの枠線カードにアイコン＋テキストで統一されているが、これではユーザーの次の行動を促せていない。

改善方針：空状態を使い分ける
- **初回空状態**: イラスト + 明確なCTA（「最初の録音をする」ボタン）
- **検索空状態**: 「条件に一致する記録はありません」+ 検索ワードのクリア提案
- **機能未実装**: 「この機能は現在準備中です」+ 代わりにできることの提案

## 4. 優先順位マトリクス

縦軸：ユーザー影響度、横軸：実装難易度

```
        高影響
          │
   ★1    │  ★2
   キャプチャ導線  │  ホーム情報設計
   状態表示      │  Ask AI 参照元リンク
          │
──────────┼──────────
          │
   ★4    │  ★3
   タスク連携    │  設定画面の整理
   空状態改善    │  文字起こし可読性
          │
        低影響
  易 ←───────────→ 難
```

**推奨実装順**: ★1 → ★2 → ★3 → ★4

## 5. 改善前後で変わること

### Before → After

| 観点 | Before | After |
|------|--------|-------|
| ホーム到着時の印象 | 未接続・検索アイコン・ファイル一覧が並び、何をすればいいか迷う | 最後に触った記録が一番上に。大きな録音ボタンが目に入る |
| 録音してから保存まで | 録音→停止→設定選択→生成開始→進捗→完了（5画面以上） | 録音→停止（自動保存、文字起こしはバックグラウンド） |
| 会議の振り返り | 要約タブを探し、縦スクロールで情報を探す | 概要タブに要点がまとまり、気になる部分は文字起こしタブですぐ確認 |
| AIに質問 | スコープを選び、質問を入力。文脈がリセットされる不安がある | 質問するだけ。参照元をタップすれば元データに飛べる |
| エラー時 | 技術的な文言が表示され、何をすればいいかわからない | 日本語で状況説明 + 再試行ボタンが表示される |
| 毎日の継続利用 | タスクは動かず、検索は使いにくく、AIは独立している | 記録→要約→タスク抽出→質問→振り返りがシームレスにつながる |
