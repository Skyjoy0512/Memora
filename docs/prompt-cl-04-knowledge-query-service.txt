# 4. 現状理解（コードから読み取った事実）

## AskAIView.swift の現状（1147行）
- `buildFileContext(fileID:)` — transcript/summary/memo/referenceTranscript を直接 FetchDescriptor で取得（line 542-587）
- `buildProjectContext(projectID:)` — プロジェクト配下ファイルの summary/memo/todo を取得（line 589-646）
- `buildGlobalContext()` — 最近ファイルの summary と未完了 todo を取得（line 648-692）
- `makePrompt(userMessage:contextPack:)` — AI プロンプト構築（line 722-745）
- `makeContextPack(scopeTitle:sources:)` — AskAIContextPack に束ねる（line 694-720）
- メモリコンテキスト: `memoryContextSources()` で MemoryFacts/MemoryProfile を追加（line 747-761）
- 全てのデータ取得が View 内の private メソッドで行われている（FetchDescriptor 直操作）

## KnowledgeIndexingService.swift の現状
- `rebuildIndex(for:)` — transcript/summary/memo/photoOCR/referenceTranscript を chunk 化
- `chunkText(_:targetLength:hardLimit:)` — ~220文字単位で分割（static メソッド）
- `extractKeywords(from:limit:)` — 正規表現でキーワード抽出（static メソッド）
- `scopeTargets(for:)` — file/project/global スコープを作成

## KnowledgeChunk.swift の現状
- scopeType: file/project/global
- sourceType: transcript/summary/memo/todo/photoOCR/referenceTranscript
- keywords: [String], rankHint: Double
- createdAt / updatedAt でソート可能

## 問題点
1. **コンテキスト構築が View に密結合** — FetchDescriptor を使った直接取得が View に散在
2. **KnowledgeChunk が使われていない** — インデックスは作成済みだが、AskAI は KnowledgeChunk を検索していない
3. **project/global の取得が粗い** — 上位4-5件を固定取得するだけでランキングがない

# 5. 実装要件

### KnowledgeQueryService（新規）
1. `@MainActor` で `ModelContext` を受け取る
2. 公開メソッド:
   - `func buildContext(for scope: ChatScope) async -> AskAIContextPack`
   - `func fetchFileContext(fileID: UUID) -> AskAIContextPack`
   - `func fetchProjectContext(projectID: UUID) -> AskAIContextPack`
   - `func fetchGlobalContext() -> AskAIContextPack`
3. file scope:
   - AudioFile から transcript/summary/memo/referenceTranscript を直接取得
   - KnowledgeChunk を sourceType でフィルタして追加コンテキストを取得（photoOCR 等）
4. project scope:
   - KnowledgeChunk を `scopeType == .project && scopeID == projectID` で検索
   - rankHint 降順で top-6 を取得
   - ファイル概要として summary 優先、なければ transcript を使用
5. global scope:
   - KnowledgeChunk を `scopeType == .global` で検索
   - rankHint 降順 + createdAt 降順で top-8 を取得
   - 未完了 todo も併用（既存ロジックを踏襲）
6. メモリコンテキスト（MemoryFacts/MemoryProfile）もこのサービスで扱う
7. `AskAIContextPack` / `AskAIContextSource` / `AskAISourceBadge` / `AskAICitation` 型は一旦 AskAIView からそのまま参照（将来的に別ファイルに移動してもよいが、このタスクでは移動しない）

### AskAIView の変更
1. `KnowledgeQueryService` をプロパティとして保持
2. `buildContextPack(for:)` を `KnowledgeQueryService.buildContext(for:)` に委譲
3. `buildFileContext / buildProjectContext / buildGlobalContext / makeContextPack / makePrompt` の private メソッドを View から削除
4. `fetchAudioFile / fetchTranscript / fetchMeetingMemo / fetchProject / fetchAudioFiles / fetchRecentAudioFiles / fetchTodos` などの FetchDescriptor ヘルパーも View から削除（KnowledgeQueryService に移動）
5. メッセージ送信時のプロンプト構築もサービス側で行う形にする

### 最小差分の方針
- `AskAIContextPack` 等の型定義は一旦 View ファイルに残す（移動は別タスク）
- KnowledgeQueryService のイニシャライザは `init(modelContext:)` のみ
- ranking は `rankHint` のみを使い（シンプル）、ベクトル検索等は導入しない
